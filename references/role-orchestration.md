# Role Orchestration

角色不是装饰，也不是每篇必备清单。Anlin 的角色系统有三层：固定熟人、低频但可验证角色、一次性称呼原型。写作时先做角色预算，再写场景。

## 二、Skill 纳入考量的常出现角色

高频不等于必须出场。常见角色只在需要其功能时出场；没有对应功能时宁可不用。

| 角色 | 优先阶段 | 适用功能 | 禁止用法 |
|---|---|---|---|
| 狗哥 | Phase A/C | 一句话刺破自欺；讲别人不敢讲的残酷真相 | 只发朋友圈、只当吐槽机、代替作者讲金句 |
| 舍友/室友 | Phase A/B/C | 同辈比较、正常人变形镜、阶级幽灵 | 每篇都拿来比惨；把舍友写成反派 |
| 我妈 | 全阶段 | 家庭温度、务实压力、生活照料 | 假设性温情：“我妈要是知道”；空泛母爱 |
| Java大哥 | Phase A | 互联网坠落后的草台班子哲学 | Phase B 以后硬塞；写成职场导师 |
| 水哥 | Phase B/C | 荒诞生活镜像、别人仍在往前走 | 只复制泻药洗虾；当普通朋友聊天 |
| 我姐 | Phase A | 暴力、关心、真相三位一体 | 只负责骂人；脱离家庭/姐弟现场 |
| 我爸/我叔 | Phase A/B 少量 | 父权毒舌 / 温暖长辈 | 同篇混用多个长辈制造角色噪音 |

### 固定角色预算

- 标准日寄：0-2 个固定角色。多数情况下 1 个够了。
- 长篇/多场景：最多 3 个固定角色，且每个应承担不同功能。
- 真诚文/身体文：可以 0 个固定角色，靠身体、记忆、陌生人完成。
- 13/38 篇原文没有任何 Tier 1 recurring cast 成员；因此“无固定熟人”本身是合法结构。
- 若固定角色出场，只能服务一个场景的转向，不能成为全文轮班表。

## 三、不固定出场角色：称呼原型与编排规则

这些角色不是 canon，可随机生成，但称呼必须像 Anlin，会来自真实生活关系而不是小说人设。

| 原型 | 原文称呼形态 | 功能 | 生成规则 |
|---|---|---|---|
| 荒谬权威 | 面试官、HR、经理、领导 | 制度荒诞的具体面孔 | 用职位称呼，不起花名；一句话暴露系统逻辑 |
| 街头智者 | 保安大爷、煎饼阿姨、盘核桃大爷 | 民间歪理/生活秩序 | “职业/动作 + 年龄称呼”，不写姓名 |
| 温柔过客 | 老奶奶、清洁阿姨、40多岁阿姨、小孩 | 让温情自然漏出来 | 不解释善良，只写动作或一句话 |
| 阶级镜像 | 北漂朋友、同学、学长、办公室小哥 | 显示别人已进入正常轨道 | 不命名；用社会位置和一个具体数字/习惯锚定 |
| 荒诞同伴 | 鬼火少年、我哥、水哥女朋友 | 生活偶然性和笑点 | 出现一次，做完动作就走 |
| 旧记忆人物 | 高中女朋友、初中女神、学姐、高三老师 | 时间破坏青春记忆 | 不复刻青春文学；必须被现实/身体/尴尬降格 |
| 超现实人物 | 芝士雪豹、蓝头发青年、下棋青年 | Phase B 特例实验 | 只在明确超现实标题/场景中使用 |

### 随机角色命名规则

Anlin 很少给一次性人物起完整姓名。生成随机人物时优先使用：

- 关系称呼：朋友、同学、学长、我哥、我妹、女朋友。
- 职位称呼：面试官、HR、经理、领导、保安、外卖小哥。
- 年龄/身份称呼：大爷、阿姨、老奶奶、小孩、小伙。
- 场景称呼：隔壁桌东北老铁、蓝头发青年、下棋青年。

禁止：

- 给路人起小说式姓名。
- 一篇里连续制造 4 个新角色。
- 让随机角色承担固定角色的功能，例如让“路人朋友”替狗哥说残酷真相。
- 让角色只出现为“有人发了张图/跟我聊了几句”。没有载荷就删。

## 四、角色编排决策树

写作前先回答：

1. 这篇是否需要固定熟人？如果主题是身体、独处、回忆、街头观察，答案可以是否。
2. 如果需要，谁的功能最不可替代？只能选 1 个主角色。
3. 是否需要第二固定角色制造对照？若只是热闹，删。
4. 随机角色是否能完成同样功能？如果能，优先随机角色，避免狗哥/水哥过劳。
5. 每个角色是否带来一句话、一个动作、一个信息？没有载荷，删。

输出前检查：

- 固定角色数是否 >2？若是，必须说明每个角色的独立功能。
- 随机角色数是否 >3？若是，删到只剩最有载荷的。
- 角色功能是否重叠？两个毒舌角色不能同时毒舌；一个负责拆台，另一个应负责现实锚定或温情。
- 角色称呼是否像 Anlin？“王先生/李小姐/陌生男子”不像；“保安大爷/隔壁桌东北老铁/办公室小哥”像。
