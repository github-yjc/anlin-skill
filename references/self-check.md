# Self Check

## 八、写后自查

自查逐条执行，不得跳过。主观判断不能替代硬规则检查；硬规则同时运行 `scripts/check_anlin_violations.py`。

### 快速诊断

先做三项；任何一项 FAIL，直接进入 §8.6 迭代，不必继续查其他项。

- [ ] **情感单频检查**：全文有笑点吗？有自嘲吗？有嘴硬吗？如果全篇只忧郁/只平淡 → FAIL。
- [ ] **高频词检查**：其实 / 觉得 / 发现 / 好像 / 不过 / 突然 / 于是 / 因为 / 所以 出现几个？如果全部 0 次 → FAIL。
- [ ] **词汇域检查**：随机扫 5 个实词——"压缩机""缓解期""抗争""冷藏室""遥控器"这类词在 Anlin 语料中存在吗？如果多数为 0 次 → FAIL。
- [ ] **AI 气味检查**：是否存在一眼能被读者指出的 AI 句式（不是X是Y、这说明、本质上、真正的问题是、平衡对照、解释性翻译）？存在 → FAIL。
- [ ] **文艺注解检查**：是否用 `——`、`那种...的放松`、`释然`、`自洽`、`真实感` 等词把画面翻译成情绪？存在 → FAIL，改成动作/身体/付款/路线/对方原话。
- [ ] **变量解释检查**：是否用 A/B、甲乙、从A到B 把日常失败抽象成模型解释？存在 → FAIL，保留具体物件和动作。
- [ ] **背景事实检查**：地名、游戏、平台、工作、学校、日期是否有语料/用户/检索依据？无依据的具体名词 → FAIL，降级为低断言表面。
- [ ] **屏幕/对话表面检查**：是否有 `底下跟了一串回答`、多人评论链、或微信/饭桌对话的独立引号行？存在 → FAIL，改成一条屏幕表面或 `问/说/回` 的行动叙述。
- [ ] **中段随机性检查**：中段是否只是继续执行题面？如果没有一个有钩子的旁逸分支 → FAIL。

以上三项全部 PASS 后，再进行下面的逐条检测。

### 8.2 声音检测

**结构检测**：观察非表演？≥3 不相关场景？蒙太奇非独幕剧？结尾未解决不升华？场景间靠联想跳跃？

**叙述者检测**：有性格反应（非摄像头式"我躺在床上听""翻了个身"）？嘴硬（明明在意说不在意，"挺好的。"）？有洞察层（不是全场景停留在"发生了什么"）？

**情感检测**：有"笑完更难受"处？情感混频（好笑+心酸+怀旧+自嘲+接受，不只忧郁）？无连续多篇金钱？

**语言检测**：无对话引号？无评论链？无"写到这"？无"不是X是Y"？无装饰性比喻？无拟人抒情？无抽象概括？

**灵魂检测（最关键）**：这篇文章读起来像"我写了一篇日寄"，还是像"我今天过了一天，顺便记了下来"？如果是前者，重写。

#### 8.2.1 灵魂门控（与 subagent-prompts.md 评分规则一致）

以下门控将主观判断转化为可复核的结构检查点。逐条执行，每条单独判定 PASS/FAIL。**新手优先检查 🔴 标注项（最常见失败），其次 🟡，最后 ⚪。**

| # | 检测项 | 阈值 | 动作 |
|---|--------|------|------|
| 🔴 22 | **情感震荡** | 至少 3 种不同情感区间（笑/刺痛/温情/嘴硬/荒诞/怀旧），且至少 1 处剧烈切换（两种对立情感出现在 7 行内） | ≤2 种 → FAIL；≥3 种但无剧烈切换 → WARN |
| 🔴 23 | **对话密度** | 至少 1 处直接引用的对话（非转述、非旁听、非省略式概括）——出现率约 66%。无对话不自动 FAIL；连续多篇无对话时检查多样性。 | 0 处→WARN；连续 3+ 篇无对话→必须有一篇加入 |
| ⚪ 24 | **元叙事** | 至少 1 处叙述者对写作行为/读者/自身的即时评论——出现率仅 30-50%，非每篇必有。 | 0 处→正常，非缺陷（50-70%的Anlin原文也没有） |
| 🟡 25 | **场景深度** | 至少 2 个场景含 ≥3 个内部节拍（节拍 = 设定→转折→深化/反转/punch）——部分短文章天然场景数少但每个场景深度够。 | ≤1 个达标→WARN；Phase A（段子密集型）可豁免 |
| 🟡 26 | **结尾重量** | 结尾句是否可互换？单独读结尾句，判断它是否承载了前文的累积重量 | 可互换（如"睡吧"无前文蓄力、"晚安""关机睡觉"）→ FAIL |
| 🟡 27 | **无力观察** | 是否存在"其实不知道在吵什么""好像也没什么区别""不就是""无非是"等空洞落点？这些句子读完后没有任何信息增量 | 存在 → FAIL（该场景删除或改写） |
| 🟡 28 | **假感动** | 是否存在"看了好一会""不知道该说什么""就那么看着"等静止情绪画面？（注："沉默了"已移除） | 存在且为独立场景收束 → FAIL |
| ⚪ 29 | **新闻名称独立观察** | 是否存在"刷到X的新闻""看到X的热搜"且个人连接仅为公式填充？ | 存在 → FAIL |
| ⚪ 30 | **脊椎** | 全文是否有贯穿性概念/意象/running gag？"今天几号？"无脊椎但每个字参与完整微型叙事。 | 无脊椎+场景互换性高 → WARN |
| ⚪ 31 | **延迟落点** | 前面埋的细节是否在后文被回收？加分项非必选项。 | 有→加分；无→正常 |
| ⚪ 32 | **自欺时刻** | 叙述者是否在某处暴露了在对自己撒谎？出现率约 50%。 | 无自欺→正常，不 FAIL |
| ⚪ 33 | **智识分析** | 是否有一处不止于观察、进入了真正的分析思考？ | 全程观察→ WARN |
| ⚪ 34 | **场景不可互换** | 单独读每个场景：放进另一天的日寄是否也能成立？ | 超半数可互换 → WARN |
| 🔴 35 | **解释癖** | 好画面后面是否有"好像""其实""突然觉得"追加哲学注解？画面自己会说话。 | 存在→删除注解句 |
| ⚪ 36 | **人工计时** | 是否存在不符合人类感知习惯的精确计时（"看了十秒"）？倒计时场景除外。 | 非倒计时秒级计时 → 违规 |
| 🟡 37 | **对话堆叠** | 是否连续三行以上以"说"开头？ | 存在→变换 |
| 🔴 38 | **单主题检测** | 全文场景是否可归为同一主题标签？日寄是碎片。 | 超半数同主题 → FAIL（重写） |
| ⚪ 39 | **单字独占分行** | 是否存在纯装饰性分行（"关门，"）？有情感重量（"甜的。"）可保留。 | 删除无损失 → 取消独占 |
| ⚪ 40 | **替别人做心理推测** | 叙述者是否在替别人"想"/"以为"？ | 存在 → 删除 |
| 🔴 41 | **二元对比** | 是否存在"不是因为X就是Y""不是X而是Y""不是X，是Y"句式？直接陈述Y或让场景呈现。 | 存在→FAIL |
| 🟡 42 | **碎片化装深刻** | 是否存在三个以上连续1-3字短句堆叠？ | 存在且无情感重量→合并 |
| ⚪ 43 | **列举过载** | 是否存在三个以上并列项？选最有意义的1个写。 | 存在→精简 |
| 🔴 44 | **背景事实漂移** | 是否出现无依据地名、城市路线、公司园区、游戏角色细节或阶段不合的 AI/GPT/平台事实？ | 存在→FAIL |
| 🔴 45 | **AI 解释腔** | 是否出现本质上/核心是/这说明/这意味着/换句话说/总之等把场景翻译成结论的句子？ | 存在→FAIL |
| 🔴 46 | **中段题面链** | 文章中段是否仍在按提示词顺序推进，而没有一个自带转向的旁逸场景？ | 存在→FAIL |
| 🟡 47 | **单句气味** | 是否有一句话删掉后全文更像真人？ | 有→删除或重写局部 |
| 🔴 48 | **文艺注解腔** | 是否用破折号或抽象情绪词给画面加字幕（`——那种...`、`放松`、`释然`、`自洽`）？ | 存在→FAIL |

**门控执行规则**：
- §8.2 的门控与 `subagent-prompts.md` 的评分规则一致（Gate 1-5 对应检测项 22-26）。
- 自查人先过 §8.1（硬规则），再派独立子代理跑 §8.2 门控。**自查人不得对自己草稿同时打 §8.1 和 §8.2 分——实现与审查必须分离。**
- **重要**：30-34 为 WARN 级（非 FAIL）——Anlin 自己的文章中，部分经典篇目（如"今天几号？"20220416）在这些项上也会 WARN。WARN 触发复核和多样性检查，但不强制重写。
- 22-29 中标记 FAIL 的项必须回修对应维度。标记 WARN 的项触发复核和多样性检查，不强制重写。
- 门控测量的是 Anlin 声音的**结构条件**，非声音本身。

### 8.2.2 修复指南（FAIL 后怎么修）

| FAIL 项 | 修复策略 |
|---|---|
| **单主题 (#38)** | 最常见失败。回到认知预备——你今天除了XX之外还碰巧看到了什么？街上的猫、邻居的声音、手机弹出来的推送、身体的不适。找一个与主题完全无关的观察，插入场景序列的中间位置。至少保证 30% 场景与主题无关。 |
| **情感震荡 (#22)** | 逐场景标注当前情感。找到情感类型最少的那一列——缺笑？插入一个荒诞观察。缺痛？插入一个身体约束。缺温情？插入一个对陌生人的具体观察（不是"我很温暖"，是描述他人行为）。至少确保三种情感各出现一次。 |
| **场景深度 (#25)** | 把单节拍场景（"我看到X→X是Y"）扩展为三节拍：设定→意外转向→深化/反转。问自己：这个场景里，读者在哪个词被颠覆了预期？如果找不到→这个场景还没有 punchline。给他一个。 |
| **对话密度 (#23)** | 找一个他人出场最多的场景，把叙述者对他人的转述（"他说他很累"）改成一句直接引用的原话（"他说跑了一天才四十七块五"）。对话必须从真实社交场景中生长——检查：现实中有人这么说吗？ |
| **无力观察 (#27)** | "其实不知道在吵什么"→删掉整句。如果场景失去这句话后没有了落点→这个场景根本没有落点，需要重建。好落点是一个具体动作或判断："我翻了个身"、"关掉了"、"甜的。" |
| **解释癖 (#35)** | 找到好画面后的注解句，删除。"镜子里的人跟二零二二年一模一样"→停，删除"好像什么都没变"。读者不需要你帮他做总结。画面就是论点。 |
| **结尾重量 (#26)** | 单独读结尾句——它能独立存活吗？如果"睡了。"前面 80 行没有任何蓄力，改成具体的身体动作或未完成状态。找前文出现过的意象（水渍、风扇、余额、脚踝）——结尾回收它但不升华它。 |
| **假感动 (#28)** | "看了好一会"→改成一个具体动作："关了，又打开。"、"划掉了。"、"放下手机。"。情绪用行为表达，不用静止画面表达。 |

### 8.3 16误区速查

1. **金句焦虑**：每段塞包袱 → 像段子合集。Anlin 大多数段落是朴素叙述，金句穿插其中。
2. **自贬过载**：每段说自己多差 → 卖惨。Anlin 更多在观察外部世界，自贬只是偶尔落点。
3. **术语堆砌**：强行插程序员术语。术语须服务当下话题，否则装腔作势。
4. **书面词汇泄漏**：然而 / 因此 / 可是 / 也许 / 或许 → 替换。
5. **口语词错位**：前两天 / 蛮 / 许多 / 啥的 → 替换。
6. **真诚泛滥**：每篇结尾走心感悟 → 真诚片段是稀缺资源，用多贬值。
7. **记忆锚点强制调用**：每篇"想当年送外卖"。锚点由场景结构性触发，非例行回顾。
8. **说教者**："所以我们应该""其实人生就是"→ 删。姿态是"我烂成这样，你自己看着办"。
9. **哲思结尾**："也不知道它们在坚持什么"→ 感官结尾替代哲学总结。
10. **场景无爆点**：每场景须有独立落点。删掉它文章失去什么 ≠ "什么都没失去"。
11. **人设崩塌**："关机睡觉""按时吃饭"→ 摆烂青年不写正常作息。
12. **假设性温情**："我妈要是知道""狗哥如果在"→ 只写实际发生的事件。
13. **平面化叙述者**：全场景停留在"发生了什么"，无一处进入分析洞察层。
14. **细节来自想象**：椅子上想的细节全删。Anlin 每个细节有真实眼睛作来源。
15. **抽象概括**："那种笑你见过很多次"→ 改为具体的可感知细节。搜"那种""你见过"→ 全改。
16. **视角错位**：用行业观察者视角写时代 → Anlin 是生活体验者，刷手机的普通人。
17. **空洞观察**：读完任何一个场景后，这个场景的落点是什么？如果落点是"其实不知道在吵什么""好像也没什么区别""不就是那点事""无非是又一天"——没有任何信息增量，没有任何笑/痛/洞察→该场景不存在。场景不是"发生了什么"的记录，而是"这个发生让我感受到了什么"的传递。
18. **假对话**：逐句读每一处对话，问"现实世界中会不会有人这样说"。如果答案是不可能——"也不枉你摆烂四年自毁前途"、"看看你会不会抽"（一个不抽烟的保安给陌生人递烟来测试对方）——删除整段对话。Anlin 的对话荒诞但不虚假。荒诞生长在真实的社交土壤里。假的去掉。
19. **新闻名称独立观察**：出现"刷到X的新闻""看到X的热搜"且后面只是公式化的"下面吵了X条，我关了"→删除整个场景。时代背景通过个人日常浏览渗透："翻裁员消息""刷到一条视频""群里在传""点开评论看到"。如果你写的句子以新闻标题开头且个人反应只是为了满足公式→删。
20. **假感动收尾**：场景以"看了好一会""沉默了""不知道该说什么""就那么看着"结尾→删除。Anlin 的情感落点是具体动作（关掉、划掉、回了一句去你妈的、又打开了），不是情绪的静止画面。静止是空的，动作是满的。

### 8.4 卡住时

重读 `references/samples-index.md`，随机选 3-5 条感受节奏、语气、幽默密度，然后继续。如果反复卡在同一类型问题上，回到 Anlin 原文完整读一篇，不只看片段。

### 8.6 迭代机制

原始编号从 §8.4 跳到 §8.6；保留该编号以兼容旧引用。

写作不是一次成型的。按以下循环执行，直到 blocking/error 问题处理完，warning 已人工复核：

1. **初稿**：完成预写流程后，按思维模型、结构规则和语言规则写出第一版。
2. **自查**：逐条执行 §8.1 高频违规检测和 §8.2 声音检测。
3. **判定**：
   - blocking/error 清零且 warning 已复核 → 进入步骤 4。
   - 发现违规 → 立即修复违规项，重新输出全文，回到步骤 2 重新自查。
   - 修复后仍有 blocking/error → 继续修复；warning 不自动阻塞，但必须记录是否采纳。
4. **输出**：只有在 blocking/error 问题处理完后，才输出最终文章。

常见迭代场景：

- 发现"不是X是Y"句式 → 删除否定铺垫，直接陈述Y，重新输出全文，重新自查。
- 发现评论链 → 删除"有人说B"，只保留氛围概括或一句代表性引用，重新输出全文。
- 发现行末句号过多 → 逐行检查，将非落点句号改为逗号，重新输出全文。
- 发现独幕剧（场景<3）→ 增加 1-2 个不相关场景用空行硬切，重新输出全文。
- 发现词汇域漂移（如代理文章 62/66 词为 0-1 次）→ 对照 §6.6 替换指南逐词替换，重新输出全文。此步不可跳过。
- 发现高频词全部缺失（其实/觉得/发现/好像/不过/突然/于是/因为/所以全部 0 次）→ 在适当位置自然插入至少 5 个，重新输出全文。
- 发现情感单频（无笑点/无自嘲/无嘴硬）→ 给至少 2 个场景增加笑点或自嘲，重新输出全文。

不要在 blocking/error 未处理的情况下输出文章。warning 是审查提示，不是自动失败。
